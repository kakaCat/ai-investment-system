## 📋 如何使用

### 查看当前任务

```bash
cat management/sprints/current.md
```

### 更新任务状态

1. 打开 `sprints/current.md`
2. 找到对应任务
3. 更新状态：
   - ⏳ 待开始 → 🔄 进行中
   - 🔄 进行中 → ✅ 完成
4. 记录完成时间
5. 更新进度统计

### 添加新任务

使用任务模板 `templates/task-template.md` 创建新任务。

### Sprint归档

Sprint结束后：
1. 将 `current.md` 移动到 `archive/sprint-{nnn}.md`
2. 创建新的 `current.md`
3. 更新当前Sprint信息

## 🔄 工作流程

```
1. 产品经理 → docs/product/prd/ 编写PRD
   ↓
2. 架构师 → docs/architecture/ 设计架构
   ↓
3. 项目经理 → management/sprints/current.md 拆分任务
   ├─ BE-001, BE-002... (后端任务)
   ├─ FE-001, FE-002... (前端任务)
   └─ QA-001, QA-002... (测试任务)
   ↓
4. 开发人员 → 领取任务 → 开发 → 更新状态
   ↓
5. 测试人员 → tests/ 编写测试 → 更新状态
   ↓
6. 项目经理 → 检查进度 → 归档Sprint
```
