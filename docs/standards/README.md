## 📐 核心原则

### 1. 根目录简洁
- ✅ 只保留 `CLAUDE.md`
- ✅ 所有文档在 `/docs`
- ✅ 脚本在 `/scripts`
- ✅ 配置在 `/config`

### 2. 金字塔原理
- **L1**: 项目根目录 + 核心索引
- **L2**: 文档类型分类（prd/design/guides）
- **L3**: 版本/功能分组
- **L4**: 具体文档
- **L5**: 文档章节（大文档拆分）

### 3. 命名规范
- 目录: `小写-短横线`
- 索引: `README.md`（必需）
- 版本: `{name}-v{n}.md`
- 功能: `{feature}-{type}.md`

### 4. 大小限制
- < 500 行: 单文件
- 500-2000 行: 考虑拆分
- \> 2000 行: 必须拆分

## 💡 使用建议

### 创建新文档前
1. 阅读 [文档管理规范](DOCUMENT-MANAGEMENT-STANDARD.md)
2. 确定文档类型和位置
3. 使用 `./scripts/docs/create-doc.sh` 创建

### 整理文档时
1. 查看 [重组总结](DOCUMENT-REORGANIZATION-SUMMARY.md) 了解历史
2. 使用 `./scripts/docs/organize-docs.sh` 整理
3. 更新相关 README 索引

### 遇到问题时
1. 参考 [全局规范](~/.claude/CLAUDE.md)
2. 查看反面示例（规范文档中）
3. 检查文档备份（`backup/` 目录）
