## 🎯 测试策略概述

本项目采用**金字塔测试策略**，确保软件质量的同时保持测试效率：

```
       /\
      /  \     E2E测试 (10%)
     /    \    - UI自动化测试
    /------\   - 端到端流程测试
   /        \
  /   集成    \  集成测试 (30%)
 /    测试     \ - API测试
/--------------\ - 数据库集成测试
/              \
/    单元测试    \ 单元测试 (60%)
/________________\ - Service/Converter/Repository
                   - 工具函数测试
```

### 核心原则

1. **测试优先**: 关键功能先写测试再实现
2. **自动化优先**: 所有可重复测试必须自动化
3. **快速反馈**: 单元测试 < 1秒，集成测试 < 10秒
4. **持续集成**: 所有测试在CI/CD中自动运行
5. **覆盖率目标**: 代码覆盖率 > 80%，关键路径 100%
