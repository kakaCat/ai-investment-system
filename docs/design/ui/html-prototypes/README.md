## 🎨 持仓操作标签体系

### 操作建议标签

Dashboard 的持仓概览表格新增了"操作建议"列，通过标签提示用户需要关注的股票：

#### 🔴 紧急操作类

**🔻 建议减仓**
- **颜色**: 红色 `bg-red-100 text-red-800`
- **触发条件**:
  - 受重大负面事件影响（如美联储加息）
  - AI 评估风险较高
  - 持仓占比过高需降低风险敞口
- **示例**: 腾讯控股（受美联储加息影响）

**🚨 立即止损**
- **颜色**: 深红色 `bg-red-200 text-red-900`
- **触发条件**:
  - 短期内大幅下跌（-10%以上）
  - 出现重大利空事件
  - AI 建议立即平仓

---

#### 🟡 关注类

**💎 加仓机会**
- **颜色**: 黄色 `bg-yellow-100 text-yellow-800`
- **触发条件**:
  - 超跌反弹机会
  - 长期价值稳定但短期下跌
  - AI 评估为买入时机
- **示例**: 青岛啤酒（-20.3%超跌，等待60元加仓）

**👀 需要关注**
- **颜色**: 橙色 `bg-orange-100 text-orange-800`
- **触发条件**:
  - 有中等影响的事件
  - 股价波动较大
  - 需要密切观察

---

#### 🟢 稳定持有类

**✓ 持有观望**
- **颜色**: 绿色 `bg-green-100 text-green-800`
- **触发条件**:
  - 表现稳定，盈利中
  - 无重大事件影响
  - AI 建议继续持有
- **示例**: 比亚迪（+14.3%盈利，表现稳定）

**🎯 已达目标**
- **颜色**: 深绿色 `bg-green-200 text-green-900`
- **触发条件**:
  - 达到预设的盈利目标
  - 建议分批止盈

---

### 事件标签

配合操作建议，显示影响该股票的事件状态：

**⚠️ 重要事件**
- **颜色**: 红色边框 `bg-red-50 border-red-200`
- **说明**: 有 Critical/High 级别事件影响
- **示例**: 腾讯（美联储加息）

**⚠️ 有事件**
- **颜色**: 橙色边框 `bg-orange-50 border-orange-200`
- **说明**: 有 Medium 级别事件
- **示例**: 青岛啤酒（财报不及预期）

**📢 利好消息**
- **颜色**: 蓝色边框 `bg-blue-50 border-blue-200`
- **说明**: 有正面事件
- **示例**: 公司回购、业绩超预期

## 🎯 标签使用场景

### 场景 1: 风险提示
```html
腾讯控股:
  🔻 建议减仓 + ⚠️ 重要事件
  → 用户一眼看出需要减仓 + 点击查看事件详情
```

### 场景 2: 加仓机会
```html
青岛啤酒:
  💎 加仓机会 + ⚠️ 有事件
  → 虽然有负面事件，但 AI 判断是加仓机会
```

### 场景 3: 稳定持有
```html
比亚迪:
  ✓ 持有观望
  → 表现稳定，无需操作
```

## 🔄 标签逻辑设计

### AI 决策流程

```
事件影响分析
    ↓
评估风险等级
    ↓
判断操作建议
    ↓
生成标签
```

### 标签优先级

当一个股票有多个标签时，按以下优先级显示：

1. **紧急操作** > 加仓机会 > 需要关注 > 持有观望
2. **重要事件** > 有事件 > 利好消息

### 标签更新频率

- **实时更新**:
  - 有新事件发生时立即更新
  - 股价大幅波动时重新评估

- **定时更新**:
  - 每日开盘前更新（基于前一日事件）
  - 每日收盘后更新（基于全天走势）

- **手动触发**:
  - 用户点击"刷新 AI 建议"
  - 用户添加/删除持仓后重新分析

---

## 💡 设计理念

1. **可操作性**: 标签要告诉用户"做什么"，而不只是"什么状态"
2. **优先级明确**: 用颜色区分紧急程度（红>黄>绿）
3. **信息密度**: 一眼看出关键信息，点击查看详情
4. **AI 驱动**: 标签基于 AI 分析，而非简单的规则
5. **事件关联**: 标签与事件系统深度集成

## 🛠 技术实现

### 前端显示
```vue
<template>
  <div class="flex gap-1">
    <!-- 主要操作建议 -->
    <span
      :class="actionBadgeClass"
      class="px-2 py-1 rounded text-xs font-semibold">
      {{ actionIcon }} {{ actionText }}
    </span>

    <!-- 事件提示（可选） -->
    <span
      v-if="hasEvent"
      :class="eventBadgeClass"
      class="px-2 py-1 rounded text-xs">
      ⚠️ {{ eventLevel }}
    </span>
  </div>
</template>
```

### 后端 API
```python
# GET /api/holdings/{account_id}/suggestions
{
  "holdings": [
    {
      "stock_code": "00700",
      "action_tag": {
        "type": "reduce",  # reduce/buy/hold/stop_loss
        "label": "建议减仓",
        "priority": "critical",  # critical/high/medium/low
        "reason": "受美联储加息影响，预期-2.5%"
      },
      "event_tag": {
        "has_event": true,
        "level": "critical",  # critical/high/medium
        "label": "重要事件",
        "count": 2
      }
    }
  ]
}
```

## 📊 数据来源

### 标签生成依据

1. **事件影响分析**
   - 从 `events` 表获取相关事件
   - AI 评估事件对持仓的影响方向和强度

2. **持仓状态**
   - 当前盈亏比例
   - 持仓占比
   - 成本价与现价

3. **市场环境**
   - 大盘走势
   - 行业趋势
   - 流动性状况

4. **用户偏好**
   - 风险偏好设置
   - 历史操作习惯
   - 资金流动性要求
